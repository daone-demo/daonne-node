# 302.AI 模型网关接口

> 目标：前端只调用 Daone 后端，后端用服务端环境变量持有 302.AI API Key。  
> API Key 不允许写入仓库、前端代码、接口文档示例或浏览器本地存储。

## 1. 统一模型变量

模型网关统一从 `appConfig.model` 读取配置：

- `local` 环境：优先读取私有文件 `config/model-provider.local.json`。
- `test` / `prod` / Vercel 环境：读取 Vercel/服务器环境变量，不读取本地私有文件。

`config/model-provider.local.json` 已加入 `.gitignore`，不会进入仓库。

可复制 `config/model-provider.local.example.json` 为 `config/model-provider.local.json`：

```json
{
  "apiKey": "sk-***",
  "openaiBaseUrl": "https://api.302.ai",
  "defaultChatModel": "gpt-5.5",
  "defaultCodexModel": "gpt-5.5",
  "defaultImageModel": "gpt-image-2",
  "toolsBaseUrl": "https://api.302.ai",
  "tools": {
    "removeBackground": "",
    "eraser": "",
    "expandImage": "",
    "increaseResolution": "",
    "imageTo3d": "",
    "promptExpert": "",
    "productImageReplacement": "",
    "poseTransfer": "",
    "topazEnhance": ""
  }
}
```

local 环境如果私有 JSON 未配置，代码会 fallback 到原有环境变量：`MODEL_API_KEY`、`MODEL_OPENAI_BASE_URL`、`MODEL_TOOL_*_PATH` 等。
local profile 默认 `DAONE_MODEL_MOCK_ENABLED=true`，不会请求 302.AI，会返回 mock 结果；需要本地真实调用时，把 `DAONE_MODEL_MOCK_ENABLED=false` 并配置私有 JSON。

Vercel 环境至少需要配置：

```env
DAONE_MODEL_MOCK_ENABLED=false
MODEL_API_KEY=sk-***
MODEL_OPENAI_BASE_URL=https://api.302.ai

SEEDANCE_API_KEY=your-seedance-api-key
SEEDANCE_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
SEEDANCE_MODEL=ark-***

HAPPY_HORSE_API_KEY=sk-ws-***
HAPPY_HORSE_BASE_URL=https://queue.fal.run
HAPPY_HORSE_TEXT_MODEL=alibaba/happy-horse/text-to-video
HAPPY_HORSE_IMAGE_MODEL=alibaba/happy-horse/image-to-video
```

## 2. 聊天 / 智能体

### 2.1 模型列表

`GET /api/v1/provider/chat/models`

返回 `/api/v1/provider/chat/completions` 可选择的模型列表：

```json
{
  "items": [
    {
      "code": "gpt5.5",
      "model": "gpt-5.5",
      "name": "GPT-5.5",
      "provider": "302.AI",
      "description": "通用智能体与复杂文本生成模型",
      "supportsStreaming": true,
      "default": true
    },
    {
      "code": "gemini-3-1-pro-preview",
      "model": "gemini-3.1-pro-preview",
      "name": "Gemini 3.1 Pro Preview",
      "provider": "302.AI",
      "description": "Gemini 3.1 Pro 预览模型",
      "supportsStreaming": true,
      "default": false
    }
  ]
}
```

前端下拉选项建议展示 `name`，提交 `/api/v1/provider/chat/completions` 时传 `code`。

### 2.2 聊天补全

`POST /api/v1/provider/chat/completions`

认证：需要 Daone 登录态 `Authorization: Bearer <Daone token>`。

请求体保持 OpenAI Chat Completions 形状：

```json
{
  "model": "gpt5.5",
  "messages": [
    {"role": "user", "content": "帮我提取这个 skill 的关键能力"}
  ],
  "stream": true
}
```

模型别名：

| 前端可传 | 转发给 302.AI |
|---|---|
| `gpt5.5` | `gpt-5.5` |
| `gpt-5.5` | `gpt-5.5` |
| `gemini-3-1-pro-preview` | `gemini-3.1-pro-preview` |
| `gemini-3.1-pro-preview` | `gemini-3.1-pro-preview` |
| `Gemini 3.1 Pro Preview` | `gemini-3.1-pro-preview` |
| `Codex` | `MODEL_DEFAULT_CODEX_MODEL`，默认 `gpt-5.5` |

当 `stream=true` 时，后端直接返回 `text/event-stream`，事件体保持 ChatGPT / OpenAI SSE 格式：

```text
data: {"id":"...","object":"chat.completion.chunk","choices":[{"delta":{"content":"..."}}]}

data: [DONE]
```

当 `stream=false` 或不传时，后端返回 302.AI/OpenAI 原始 JSON，不包 Daone 统一响应，不在 body 中追加 `traceId`。

## 3. 图片生成

`POST /api/v1/provider/images/generations`

请求体保持 OpenAI Images 形状：

```json
{
  "model": "image2.0",
  "prompt": "白底运动鞋商品主图",
  "n": 2,
  "size": "1024x1024"
}
```

模型别名：

| 前端可传 | 转发给 302.AI |
|---|---|
| `image2.0` | `gpt-image-2` |
| `gpt-image-2` | `gpt-image-2` |
| `Nanobanana 2.0` / `nanobanana-2.0` | `gemini-3.1-flash-image-preview` |
| `gemini-3.1-flash-image-preview` | `gemini-3.1-flash-image-preview` |

如 302.AI 对该图片模型支持 `stream=true`，后端同样透传 SSE，不包统一响应。
非流式图片生成同样返回 302.AI/OpenAI 原始 JSON，不包 Daone 统一响应。

## 4. 视频生成

`POST /api/v1/provider/videos/generations`

这是单独的视频模型入口。同一个接口通过 `model` 切换官方平台：

| 前端可传 | 官方平台 | 后端配置 |
|---|---|---|
| `seedance2.0` | BytePlus/火山方舟 ModelArk Seedance 2.0 | `SEEDANCE_*` |
| `happy-horse` | fal.ai HappyHorse 1.0 | `HAPPY_HORSE_*`；传 `imageUrl` 时默认走 image-to-video，否则走 text-to-video |

通用请求：

```json
{
  "model": "seedance2.0",
  "prompt": "商品鞋在白色摄影棚中旋转展示",
  "imageUrl": "https://example.com/product.png",
  "duration": 5,
  "aspectRatio": "16:9",
  "resolution": "720p",
  "stream": true
}
```

如果需要使用官方平台新增或特殊字段，可以传 `providerBody`，后端会按所选模型直接转发：

```json
{
  "model": "happy-horse",
  "stream": true,
  "providerBody": {
    "prompt": "A product video with smooth camera movement"
  }
}
```

视频接口默认使用流式响应；当 `stream` 不传或 `stream=true` 时，后端把“提交任务 + 查询状态”包装为 ChatGPT/OpenAI 风格 SSE：

```text
data: {"id":"...","object":"video.generation.chunk","choices":[{"delta":{"status":"running","progress":50}}]}

data: {"id":"...","object":"video.generation.chunk","choices":[{"delta":{"status":"succeeded","result":{"videoUrl":"https://..."}},"finish_reason":"succeeded"}]}

data: [DONE]
```

仅当显式传 `stream=false` 时，接口返回标准化 JSON，包含 `providerTaskId` 和官方平台原始返回 `raw`；前端可用于后续接入任务页或日志。

## 5. 小工具

查询工具白名单：

`GET /api/v1/provider/tools`

调用工具：

`POST /api/v1/provider/tools/:toolCode`

支持的 `toolCode`：

| toolCode | 工具 |
|---|---|
| `remove-background` | 抠图 / remove background |
| `eraser` | Eraser / 物体擦除 |
| `expand-image` | Expand Image / 图片扩展 |
| `increase-resolution` | Increase Resolution / 图片放大 |
| `image-to-3d` | Tripo3D V2.0 / 图片转 3D |
| `prompt-expert` | AI 提示词专家 2.0 / 反推提示词 |
| `product-image-replacement` | Product Image Replacement / 商品图替换 |
| `pose-transfer` | Pose-Transfer / 人物姿态变换 |
| `topaz-enhance` | Topazlabs / 画质增强 |

工具请求体由对应 302.AI / Apifox 工具文档决定，后端做白名单代理，不允许前端传任意 URL。真实环境中若对应 `MODEL_TOOL_*_PATH` 未配置，会返回：

```json
{
  "code": "MODEL_TOOL_NOT_CONFIGURED",
  "message": "模型工具未配置或不支持"
}
```

## 6. 与原生成任务接口的关系

- `/api/v1/generation-tasks`：Daone 业务任务接口，包含积分、任务状态、结果素材入库，适合画布节点生成闭环。
- `/api/v1/provider/...`：模型网关接口，主要用于前端需要 OpenAI/ChatGPT 兼容返回，尤其是 SSE 流式响应；默认不做积分扣减和素材入库。
